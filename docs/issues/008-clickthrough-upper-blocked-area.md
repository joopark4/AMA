# Issue #008: 아바타 상단 클릭스루 비통과 영역(거울형 오검출) 수정

> 작성일: 2026-02-17

## 문제 요약

투명 오버레이에서 아바타/버튼 영역이 아닌 아바타 상단 일부에서도 마우스 입력이 통과하지 않는 구간이 발생했다.

## 증상

- 아바타 상단에 보이지 않는 입력 차단 구간이 생김
- 해당 구간에서는 뒤쪽 앱 클릭이 되지 않음
- 아바타 근처가 아닌 영역에서도 간헐적으로 상호작용 모드가 유지됨

## 원인 분석

`useClickThrough`의 커서 좌표 후보 생성에서 Y축 반전 후보를 상시 포함하고 있었다.

- 정상 좌표 후보 + 반전 좌표 후보를 동시에 검사
- 반전 좌표가 `interactionBounds`와 매칭되면서 상단에 거울형 인터랙티브 영역이 생김
- 결과적으로 클릭스루가 켜져야 할 영역에서 꺼진 상태가 유지됨

## 해결 내용

- `buildCursorCandidates()`에서 상시 Y축 반전 후보를 제거
- 논리 좌표/물리 좌표(DPR) 기반 후보만 유지
- 기존 fail-safe(좌표 이상 시 interactive 유지)는 유지

## 검증

- `npx tsc --noEmit` 통과
- `npm run tauri dev` 실행 후 상단 클릭 통과 수동 확인
- 아바타/버튼/설정 패널은 기존처럼 클릭 가능 유지
